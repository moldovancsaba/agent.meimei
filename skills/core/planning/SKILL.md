# Planning

## When to use

Use this skill when work is too broad to execute safely in one pass.

## What it does

- Splits work into deliverable steps.
- Identifies dependencies.
- Defines a first actionable move.

## Instructions

- Pick the critical path.
- Separate blocking work from side work.
- Keep plans short enough to execute.

## Guardrails

- Do not build a plan that hides uncertainty.
- Do not expand scope just because it is interesting.
